The toolbar

The toolbar gives access to plexo annotation functions. It's design is minimalistic so the user can focus on the annotation task at hand instead of having to learn another complicated tool. Some tools such as the brush and the eraser have additional settings that can be accessed by double-clicking or double-tapping(in case of a touch screen).

  • Brush: selects the current label to annotate. Double click for more options
  • Bucket: this is a paint bucket. It will flood a closed region with the current label
  • Eraser: Removes annotations from the current slice. Double-click for options
  • Zoom/Panning: Toggles between zooming and panning operations on an touch screen. Not required when a mouse is available, since the mouse wheel acts as a zooming mechanism (as it is common in many graphical apps)
  • Undo/Redo: Everytime that an annotation, deletion or paint bucket operation occurs, plexo saves the state of the annotations for the current slice. The undo/redo options allow retrieving previous annotation states (undo) or repeating them (redo).
  • Propagate annotations: it allows copying one or more annotations from one slide to a range of slides
  • Download/Upload annotations: allows saving/retrieving your annotations using zip files.
  • Toolbar help: Quick tour of the toolbar using tooltips
  • Slice selection: This slider allows navigating quickly through a dataset

Using the brush

The paint brush allows selecting the current label to use for annotating an image. This annotation tool is used as in any other graphical interface to draw strokes on top of the image, thus creating annotations.

The paint brush is activated by clicking/tapping on the button. When selected for the first time, the first label in the active palette will be selected. The selected label appears in the status bar, as shown in the following figure:

Defining your own label set (Loading Labels)

A customized labelset can be specified in a JSON format as shown in the snippet below:

			
				{
					"labels": [
						{
							"id": 1,
							"name": "transverse process",
							"color": "#4986e7"
						},

						{
							"id": 2,
							"name": "spinous process",
							"color": "#a47ae2"
						},

						...

					]
				}

The expected format for customized labels in JSON is an object that contains a list named labels. Each element of this list is an object with three attributes: id, name, and color. The color of the label is specified in hexadecimal format.
